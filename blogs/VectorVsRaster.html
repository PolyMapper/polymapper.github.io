Vector vs Raster Data Models

January 5, 2019 by Gordon McLachlan

There are two main types of data models available to GIS: vector and raster. Vector models are stored through individual entities in the form of points, lines, and polygons, with raster comprising grid cells of a predefined resolution. Within current literature there is debate about the utility of vector compared with that of raster, with some asserting that vector’s accuracy is superior, and more appealing because its graphic similarity to paper maps. However, any comparison between models should be made based on utility.

Differences between the models become apparent when conducting fundamental spatial operations. Vector models calculate Euclidean distance of lines and perimeters using Pythagorean theory (Heywood et al., 2011), based on explicit coordinate locations for each node. Area is calculated using geometry, and is computed by splitting polygons into smaller, less complicated shapes, enabling swift and efficient calculation. Coordinates give vector data precision and high positional accuracy, creating distance and area calculations with veracity. Such calculations only need to be run once, with information being stored in attributable tables.

Raster distance and area calculations are simple and quick to run. Distance calculations use Euclidean distance, adding the number of cells in a straight line together and multiplying by the resolution. However, distance can also be calculated using Manhattan distance, following the length of the path only using cell side lines, illustrated in Figure 1. In addition, raster structure allows distances to be calculated from one cell to all other cells, a process called proximity. Area can be calculated by adding together the number of cells in a region, then multiplying by the resolution. However, built in resolution levels can mask geographical details, leading to underestimated results.

Overlay analysis, in a vector model, is a process of placing a point, line or polygon on top of another map layer. Overlaid features can be combined using Boolean logic to get all overlaying features (union) (Figure 2), only features that are spatially coincidental (intersection) (Figure 3), and only features that are not spatially coincidental (symmetrical Difference) (Figure 4). Conducting an overlay with vector data may seem theoretically simple but it is operationally complex. Underlying calculations determine the number of intersections occurring when drawing lines between entities and points outwith the polygon boundary (Figure 5). Polygon in polygon overlay can expose issues. If polygon boundaries are not coincidental small polygons, called slithers, can be created at polygon boundaries, and produce abnormal results.

Boolean logic can similarly be used in Raster overlay (Figures 2-4), however raster models can also combine maps using various mathematical operations, using map algebra. Map algebra is a flexible and computationally quick overlay method. Issues can arise, however, when combining raster maps of different resolutions, although using appropriate resolution reduces unnecessary data approximation.

In a similar operational process to overlay, buffer analysis allows spatial questions to be asked about an entity’s neighbours. Vector model buffer analysis is conducted by overlaying a point, line, or polygon on a fixed or varied distance buffer polygon. As with overlay analysis is it complex and time consuming, depending on data volumes.

The raster model buffer analysis is quick, with the user selecting cells around the entity. However, it is often run in a number of steps; running a proximity analysis on the entity then reclassifying the result to produce a buffer of a designated distance. Reclassification allows for friction values to be added, making the buffer spread more flexible.

Vector data modelling is dominant, having a high degree of spatial precision, and is not affected by issues of varying scale, but can be computationally complex. Raster data models have quick processing speeds and have some unique and powerful tools, however, predefined resolutions reduce potential accuracy. Ultimately the appropriate data model depends on the analysis being conducted.
